## Project

Windows-first Wails v2.11.0 desktop app (Go 1.25.5 + React 18 + TypeScript + Vite) that manages multiple Codex `auth.json` profiles and switches the active account in-place.

## Commands

- `wails dev` тАФ run the desktop app with live frontend rebuild.
- `wails build` тАФ produce `build/bin/codex-profile-manager.exe`.
- `go test ./...` тАФ run all Go tests. Single package: `go test ./internal/profile/...`. Single test: `go test ./internal/profile -run TestXxx`.
- `cd frontend && npm install` тАФ install UI deps.
- `cd frontend && npm run build` тАФ tsc + vite production build (also runs during `wails build`).
- Wails bindings are generated into `frontend/wailsjs/` тАФ do not hand-edit; regenerated by `wails dev`/`wails build`.

## Architecture

The backend is a set of domain services composed in `app.go` (`NewApp`) and bound to the frontend via Wails. The `App` struct exposes the public RPC surface; DTOs live in `api_types.go` and mapping helpers convert internal records to/from DTOs.

Service composition (dependency direction):
`paths` тЖТ `config` тЖТ `profile` + `audit` + `detector` тЖТ `switcher`. The tray controller (`tray.go`) holds a back-reference to `App` for quick switching; disabled via env (`trayDisabledByEnv`).

Key internal packages under `internal/`:
- `paths` тАФ resolves AppData locations (settings, profiles index, audit log).
- `config` тАФ reads/writes user `Settings` (includes `TargetAuthPath`, `ActiveProfileID`).
- `profile` тАФ CRUD, import (bytes/raw/file), reorder; persists to profiles index JSON. `Mode` distinguishes official vs API Key profiles.
- `switcher` тАФ the core write flow: atomic replace of target `auth.json`, updates active profile id, writes audit entry, and on cross-provider switches triggers rollout-session sync.
- `codexsession` тАФ scans `~/.codex/sessions/` and clones rollout JSONL when switching between `openai` (official) and `custom` (API key) providers, rewriting `session id`, `model_provider`, `cloned_from`, `original_provider`, `root_session_id`, `clone_timestamp`. Dedupes by `root_session_id` to avoid repeated cloning on back-and-forth switches.
- `codexcfg` тАФ maintains a managed block in `.codex/config.toml` used by API Key mode.
- `detector` тАФ identifies which profile (if any) is currently active in the target `auth.json`, reports "managed" status, runs diagnostics.
- `audit` тАФ append-only JSONL log (capped reads, last 100).
- `fsx` тАФ atomic file writes and filesystem helpers.
- `process`, `util`, `vault`, `backup` тАФ small helpers; `vault`/`backup` are present but the app intentionally does no encryption or backups (see README).

State-change flow: any mutating RPC calls `notifyStateChanged(reason)`, which emits a Wails `state:changed` event to the UI and refreshes the tray menu. `pinCurrentActiveProfile` lazily records the currently-active profile before imports so the UI does not lose "active" tracking.

Frontend (`frontend/src`) is a React + TypeScript SPA that consumes generated Wails bindings and listens for `state:changed`; theming uses CSS custom properties in `App.css` (extend existing tokens before adding new colors).

## Conventions

- Go: gofmt, tabs, exported PascalCase, internal camelCase. Keep services small and domain-scoped; avoid cross-package globals.
- TS/React: functional components, 2-space indent, camelCase.
- Tests live next to code as `*_test.go` (e.g. `internal/profile/service_test.go`).
- Profiles are stored in **plaintext** by design тАФ never commit real `auth.json` content or AppData files; test with sanitized data.
- No encryption, no backups, no cloud sync тАФ do not add these without explicit request.
